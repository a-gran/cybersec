# Техническое задание
## Игровой тренажёр «Кибердетектив: Распознай скам»

---

## 1. Общее описание проекта

Консольная игра на Python, обучающая пользователей распознавать мошеннические (скам) сообщения в интернете.

Игрок читает сообщения — будто он их получил в чате. Его задача: определить, это скам или обычное безопасное сообщение. За каждый правильный ответ начисляются очки. За ошибку — теряется жизнь. В конце игры показывается итог и звание.

**Запуск игры:**
```
python main.py
```

---

## 2. Структура проекта

Проект состоит ровно из **7 модулей**. Каждый модуль отвечает за свою часть задачи:

```
project/
├── data.py       — полная база сообщений (data) и случайная выборка из 20 кейсов (cases)
├── config.py     — константы и настройки игры
├── rules.py      — отображение правил в начале игры
├── question.py   — показ вопроса и получение ответа игрока
├── answer.py     — проверка ответа и определение итогового звания
├── results.py    — итоговый экран после окончания игры
└── main.py       — ядро: игровой цикл и запуск всех модулей
```

> Нельзя объединять модули или переносить функции между ними.
> Каждый модуль импортирует только то, что ему нужно.

## 4. Модуль `data.py` — база сообщений

### Назначение
Хранит все сообщения и формирует выборку для текущей игры. Импортирует только `random`, функций не содержит.

### Что нужно реализовать

**Переменная `data`** — полный список всех сообщений (основная база данных). Каждый запуск игры берёт из неё случайную выборку.

**Переменная `cases`** — список из **20 случайных кейсов**, выбранных из `data` без повторений:

```python
cases = random.sample(data, 20)
```

> `random.sample()` гарантирует, что одно и то же сообщение не повторится дважды в одной игре.

Каждый словарь в `data` описывает одно сообщение:

| Ключ | Тип | Описание |
|---|---|---|
| `'text'` | `str` | Текст сообщения, которое видит игрок |
| `'is_scam'` | `bool` | `True` — это скам, `False` — безопасное |
| `'explanation'` | `str` | Объяснение, почему это скам или почему безопасно |

### Требования к содержимому

- Минимум **100 сообщений** в списке
- Примерно **половина** должна быть скамом, половина — безопасными
- Тематика скам-сообщений: просьба пароля, ссылки на сторонние сайты, выигрыш приза, срочность, угрозы, просьба CVV
- Тематика безопасных: игровые приглашения, напоминания, обычные вопросы

### Пример одной записи

```python
{
    'text': 'Привет! Хочешь сыграть вместе в Minecraft?',
    'is_scam': False,
    'explanation': 'Безопасно. Обычное приглашение поиграть. Нет ссылок и просьб о данных.'
}
```

## 5. Модуль `config.py` — константы

### Назначение
Хранит все настройки игры в одном месте. Если нужно изменить, например, количество жизней — меняется только здесь.

### Что нужно реализовать

| Имя | Тип | Значение | Описание |
|---|---|---|---|
| `LIVES` | `int` | `3` | Количество жизней игрока в начале |
| `POINTS` | `int` | `10` | Очков за один правильный ответ |
| `SCAM_SIGNS` | `list[str]` | см. ниже | Список признаков скама для итогового экрана |

Список `SCAM_SIGNS` должен содержать **не менее 5 строк** — признаков, по которым можно распознать скам. Примеры признаков:
- просят пароль или код подтверждения
- слишком выгодное предложение
- искусственная срочность
- подозрительные ссылки
- просят данные карты

### Важно
Этот модуль **ничего не импортирует** и не содержит функций — только переменные.

1. Вывести строку-статус: номер вопроса, общее количество, жизни, счёт

**Пример вывода:**

#### Функция `get_choice()`

**Принимает:** ничего

**Возвращает:** `bool` — `True` если игрок считает, что это скам; `False` если считает безопасным

**Что должна делать:**
1. Запросить у игрока ввод через `input()`
2. Если введено `'1'` — вернуть `True`
3. Если введено `'2'` — вернуть `False`
4. Если введено что-то другое — вывести подсказку и спросить снова
5. Функция не завершается, пока не получит корректный ответ (`'1'` или `'2'`)

> Использовать `.strip()` при чтении ввода, чтобы пробелы по краям не мешали.

## 8. Модуль `answer.py` — проверка ответа и итоговое звание

### Назначение
Содержит логику проверки ответа и определения звания по итогам игры.

### Что нужно реализовать

#### Импорты
```python
from config import POINTS
```

---

#### Функция `process_answer(player, case, lives, score, correct)`

**Параметры:**

| Параметр | Тип | Описание |
|---|---|---|
| `player` | `bool` | Ответ игрока (`True` = скам, `False` = безопасно) |
| `case` | `dict` | Словарь вопроса, содержит `case['is_scam']` и `case['explanation']` |
| `lives` | `int` | Текущее количество жизней |
| `score` | `int` | Текущий счёт |
| `correct` | `int` | Текущее количество правильных ответов |

**Возвращает:** кортеж `(lives, score, correct)` — обновлённые значения

**Что должна делать:**
1. Сравнить `player` с `case['is_scam']`
2. Если совпадает — вывести «ПРАВИЛЬНО!», прибавить `POINTS` к счёту, увеличить `correct` на 1
3. Если не совпадает — вывести «НЕПРАВИЛЬНО!», уменьшить `lives` на 1
4. В обоих случаях вывести объяснение из `case['explanation']`
5. Вернуть обновлённые `lives`, `score`, `correct`

> Исходные переменные не изменяются снаружи — функция возвращает новые значения.

---

#### Функция `final_title(score, total)`

**Параметры:**

| Параметр | Тип | Описание |
|---|---|---|
| `score` | `int` | Итоговый счёт игрока |
| `total` | `int` | Количество вопросов в игре |

**Возвращает:** `str` — текст звания

**Логика определения звания:**

| Доля от максимума | Звание |
|---|---|
| 80% и выше | «КИБЕРДЕТЕКТИВ — отличный результат!» |
| от 50% до 79% | «ОСТОРОЖНЫЙ ПОЛЬЗОВАТЕЛЬ — неплохо, но есть куда расти.» |
| ниже 50% | «НОВИЧОК — не расстраивайся, теперь ты знаешь больше!» |

> Максимальный счёт = `total * POINTS`. Доля = `score / (total * POINTS)`.

| `score` | `int` | Итоговый счёт |
| `lives` | `int` | Сколько жизней осталось |
| `total` | `int` | Сколько вопросов было всего |

## 10. Модуль `main.py` — ядро игры

### Назначение
Точка входа. Импортирует все модули, запускает игровой цикл и связывает все части игры вместе.

### Что нужно реализовать

#### Импорты
```python
import random
import data
from config import LIVES
from rules import show_rules
from question import show_question, get_choice
from answer import process_answer
from results import show_results
```

---

#### Функция `game_loop()`

**Принимает:** ничего

**Возвращает:** кортеж `(correct, score, lives, total)`

**Что должна делать:**
1. Скопировать список `data.cases` в новую переменную (через `[:]`), чтобы не менять оригинал
2. Перемешать копию через `random.shuffle()`
3. Запомнить общее количество вопросов в переменной `total`
4. Задать начальные значения: `lives = LIVES`, `score = 0`, `correct = 0`, `question_num = 0`
5. Перебрать все вопросы в цикле `for case in shuffled:`
   - Увеличить `question_num` на 1
   - Вызвать `show_question(...)` для отображения вопроса
   - Вызвать `get_choice()` и сохранить ответ игрока
   - Вызвать `process_answer(...)` и обновить `lives`, `score`, `correct`
   - Если `lives == 0` — вывести сообщение о конце и выйти из цикла через `break`
   - Иначе — дождаться нажатия Enter перед следующим вопросом
6. Вернуть `correct, score, lives, total`

---

#### Функция `play()`

**Принимает:** ничего

**Возвращает:** ничего

**Что должна делать** (только вызовы других функций, никакой логики):
1. Вызвать `show_rules()`
2. Вызвать `game_loop()` и сохранить результат
3. Вызвать `show_results(...)` с полученными результатами

---

#### Запуск

Последняя строка файла:
```python

```
